create_sampled_pixel_coords_image¶

ivy_vision.create_sampled_pixel_coords_image(image_dims, samples_per_dim, batch_shape=None, normalized=False, randomize=True, dev_str=None)[source]¶

Create image of randomly sampled homogeneous integer \(xy\) pixel co-ordinates \(\mathbf{X}\in\mathbb{Z}^{h×w×3}\), stored as floating point values. The origin is at the top-left corner of the image, with \(+x\) rightwards, and \(+y\) downwards. The final homogeneous dimension are all ones. In subsequent use of this image, the depth of each pixel can be represented using this same homogeneous representation, by simply scaling each 3-vector by the depth value. The final dimension therefore always holds the depth value, while the former two dimensions hold depth scaled pixel co-ordinates.

[reference] deduction from top of page 154, section 6.1, equation 6.1

Parameters

  • image_dims (sequence of ints.) – Image dimensions.

  • samples_per_dim (sequence of ints.) – The number of samples to perform per image dimension.

  • batch_shape (sequence of ints, optional) – Shape of batch. Assumed no batch dimensions if None.

  • normalized (bool, optional) – Whether to normalize x-y pixel co-ordinates to the range 0-1. Default is False.

  • randomize (bool, optional) – Whether to randomize the sampled co-ordiantes within their window. Default is True.

  • dev_str (str, optional) – device on which to create the array ‘cuda:0’, ‘cuda:1’, ‘cpu’ etc.

Returns

Image of homogeneous pixel co-ordinates [batch_shape,height,width,3]
