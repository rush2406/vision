closest_mutual_points_along_two_skew_rays¶

ivy_vision.closest_mutual_points_along_two_skew_rays(camera_centers, world_ray_vectors, batch_shape=None, image_dims=None, dev_str=None)[source]¶

Compute closest mutual homogeneous co-ordinates \(\mathbf{x}_{1,i,j}\in\mathbb{R}^{4}\) and \(\mathbf{x}_{2,i,j}\in\mathbb{R}^{4}\) along two world-centric rays \(\overset{\sim}{\mathbf{C}_1} + λ_1\mathbf{rv}_{1,i,j}\) and \(\overset{\sim}{\mathbf{C}_2} + λ_2\mathbf{rv}_{2,i,j}\), for each index aligned pixel between two world-centric ray vector images \(\mathbf{RV}_1\in\mathbb{R}^{h×w×3}\) and \(\mathbf{RV}_2\in\mathbb{R}^{h×w×3}\). The function returns two images of closest mutual homogeneous co-ordinates \(\mathbf{X}_1\in\mathbb{R}^{h×w×4}\) and \(\mathbf{X}_2\in\mathbb{R}^{h×w×4}\), concatenated together into a single array.

[reference] second answer in forum

Parameters

  • camera_centers (array) – Camera center [batch_shape,2,3,1]

  • world_ray_vectors (array) – World ray vectors [batch_shape,2,h,w,3]

  • batch_shape (sequence of ints, optional) – Shape of batch. Inferred from inputs if None.

  • image_dims (sequence of ints, optional) – Image dimensions. Inferred from inputs in None.

  • dev_str (str, optional) – device on which to create the array ‘cuda:0’, ‘cuda:1’, ‘cpu’ etc. Same as x if None.

Returns

Closest mutual points image [batch_shape,2,h,w,4]
